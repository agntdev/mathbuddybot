# fix-121444c7216d60fc — Duplicate `/help` handler causes double replies

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 MATHB

`src/handlers/help.ts` and `src/handlers/T01.ts` are byte-for-byte identical files that both register the `/help` command on a `Composer`. `buildBot()` auto-loads every file in `src/handlers/`, so both composers match on `/help` and each fires `ctx.reply()`. The user receives **two identical messages** for every `/help`. This is a real functional defect — one of the two files should be removed.

Additionally, `tests/specs/T01.json` and `tests/specs/T03.json` are identical `/help` specs, and `tests/commands/T01.json` / `tests/commands/T03.json` are duplicate command declarations for `/help`. These test artifacts likely mask the duplicate-handler bug because the specs use subsequence matching — finding *one* matching `sendMessage` among the two emitted calls makes the step green, so the gate stays green despite the user-facing double-reply.
